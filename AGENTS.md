# AGENTS.md

This file provides guidance to coding agents working with code in this repository.

## Project Overview

Uniflow is an open-source Customer Data Platform (CDP) for AWS. It ingests customer events (track, identify, page, screen, group — Segment-compatible), resolves identity, stores profiles in DynamoDB, and fans out to destination connectors.

## Monorepo Structure

TypeScript monorepo managed with **pnpm workspaces** and **Turborepo**. Requires **Node.js ≥20** and **pnpm ≥9**.

| Workspace | Purpose |
|---|---|
| `infra/` | AWS CDK stack & constructs (API Gateway → Lambda → Kinesis → DynamoDB pipeline) |
| `services/ingest` | Lambda: validates events (Zod), enriches, writes to Kinesis |
| `services/processor` | Lambda: Kinesis consumer → identity resolution → DynamoDB + SQS fan-out |
| `services/connector` | Lambda: SQS consumer → runs destination connectors |
| `services/audience-builder` | Glue PySpark job: segment evaluation → S3 Parquet + DynamoDB |
| `services/management-api` | Lambda: CRUD for sources, destinations, segments, profiles |
| `services/authorizer` | Lambda authorizer for write-key validation |
| `libs/event-schema` | Zod schemas for all event types |
| `libs/identity` | Identity resolution & profile upsert logic |
| `libs/logger` | Structured JSON logger |
| `connectors/sdk` | Abstract base class for destination connectors |
| `connectors/webhook` | Webhook destination connector |
| `connectors/s3-export` | S3 export destination connector |
| `cli/` | CLI tool (`uniflow init`, `deploy`, `status`, `destroy`) using Commander.js |
| `ui/` | Next.js 15 + React 19 admin dashboard (Tailwind CSS v4, Cognito auth) |
| `sdk/js` | Browser/Node.js client SDK |
| `sdk/python` | Python client SDK |

## Common Commands

```bash
pnpm build          # Build all packages (turbo)
pnpm test           # Run all tests (turbo)
pnpm lint           # ESLint all packages (turbo)
pnpm typecheck      # TypeScript check all packages (turbo)
pnpm format         # Prettier format all files

# Single package
pnpm --filter @uniflow/service-ingest test
pnpm --filter @uniflow/cdk build

# Single test file
pnpm vitest run services/ingest/src/index.test.ts

# UI development
pnpm --filter @uniflow/ui dev
```

## Architecture

```
Client SDK → API Gateway → Ingest Lambda → Kinesis → Processor Lambda → DynamoDB
                                              ↓                            ↓
                                          Firehose → S3              SQS → Connector Lambda
                                              ↓
EventBridge Scheduler → Glue PySpark Job → S3 Parquet + DynamoDB segment members

Cognito → API Gateway → Management API Lambda
Next.js static → S3 → CloudFront (Admin UI)
```

## Key Conventions

- **Event schemas**: Zod validation in `libs/event-schema`. All events require `anonymousId` or `userId`.
- **Lambda handlers**: Export a `handler` function. AWS SDK clients initialized at module scope.
- **CDK constructs**: Each infra concern is a separate Construct class composed in `UnifowStack`.
- **Testing**: Vitest with globals enabled. Tests co-located as `*.test.ts`. AWS mocking via `aws-sdk-client-mock`.
- **Workspace deps**: Use `"workspace:*"` in package.json for internal references.
- **DynamoDB multi-table**: 6 dedicated tables — `profilesTable` (userId/sortKey), `identityTable` (anonymousId), `sourcesTable` (id, GSI `writeKeyHashIndex`), `destinationsTable` (id), `segmentsTable` (id), `segmentMembersTable` (segmentId/userId).
- **Config file**: `uniflow.config.yaml` generated by `uniflow init`, consumed by CLI deploy.

## Code Style

- Prettier: single quotes, trailing commas, 100 char width, 2-space indent
- ESLint flat config: `no-explicit-any` warn, unused vars error (prefix `_` to ignore)
- TypeScript strict mode, target ES2022, module NodeNext

## Pre-commit Hooks

Husky runs: lint-staged (ESLint on `*.{ts,tsx}`) → typecheck → test
